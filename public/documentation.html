myFlix API Documentation

This API provides users access to information about different movies, directors, and genres. Users can sign up, update their personal information, and create a list of their favorite movies on myFlix.

Business Logic Endpoint URL HTTP Method Query Parameters Request body data format Response body data format
Get a list of all movies /movies GET none none A JSON object holding a list of all movies
Get data about a movie by title /movies/:Title GET Title none A JSON object holding data (title, description, genre, director, image URL, featured or not) about the specified movie title
Get data about a genre by name (e.g., "Thriller") /movies/genres/:genreName GET genreName none A JSON object holding the name and description of the specified genre
Get data about a director by name /movies/directors/:directorName GET directorName none A JSON object holding data (name, bio, birth year, death year) of the specified director
Get a list of all users /users GET none none A JSON object holding a list of all users
Get a user by Username /users/:Username GET Username none A JSON object of the specified user
Add a new user /users POST none A JSON object holding data about the user to add, structured like: { "Username": "Jordan", "Password": "5678", "Email": "jordanlazan@yahoo.com", "Birthday": "1996-04-12" "FavoriteMovies": [] } A JSON object holding data about the user that was added, or a message that the user already exists
Update a user's info /users/:Username PUT Username A JSON object holding data about the user's updated info, structured like: { "Username": "JordanLazan", "Password": "5678", "Email": "jordanlazan@yahoo.com", "Birthday": "1996-04-12" "FavoriteMovies": [] } A JSON object holding data about the updated user
Add a movie to user's list of favorites /users/:Username/movies/:MovieID POST Username, MovieID none A JSON object of the updated user
Remove a movie from user's list of favorites /users/:Username/movies/:MovieID DELETE Username, movieID none A JSON object of the updated user
Remove a user /users/:Username DELETE Username none A text message indicating that the user was successfully deleted, or that the user was not found
